# Gamify Artifacts — UI-экраны

Этот документ фиксирует, какие экраны есть в системе и что на них видит пользователь. Здесь нет пиксельных макетов, цветов и точных компонентов UI-библиотеки — только структура экранов, их назначение и ключевые элементы. Это каркас, который поможет проектировать интерфейс осознанно и не упустить важные блоки.

Документ будет уточняться по мере того, как система начнёт использоваться в реальной игре и станет понятно, какие экраны действительно нужны, а какие можно объединить или убрать.

---

## Экран: Главная панель (Dashboard)

### Назначение

Это первый экран, который игрок видит при входе в систему. Его задача — дать мгновенный снимок текущего состояния: сколько валюты на балансе, по каким артефактам идёт прогресс, сколько сделано сегодня. Игрок должен сразу понять, что он не стоит на месте, а продвигается по своим целям.

### Основные элементы

- **Карточка игрока** — имя, аватар, текущий баланс валюты.
- **Список активных артефактов** — каждый артефакт показан как карточка с:
  - названием и категорией;
  - прогресс-баром общего прогресса (0–100%);
  - меткой прогресса за текущий день (отдельным цветом или значком);
  - оставшимся временем до дедлайна (например, «осталось 3 дня»).
- **Общий снимок прогресса** — краткая сводка: сколько артефактов активно, сколько завершено, сколько просрочено.
- **Прогресс-бары по сферам VISION** — агрегация из OKR (4 сферы). Прогресс по сферам VISION на Dashboard показывает **текущий** агрегированный прогресс связанных OKR за выбранный период (например, квартал). В отличие от артефактов, здесь не выделяется отдельный показатель «за сегодня»: прогресс сфер отображает общую динамику выполнения целей, а не ежедневный прирост.
- **Быстрые ссылки** — кнопки для перехода в магазин, инвентарь, коллекцию артефактов, OKR, VISION.

> **Источник данных:** агрегированное API (например, `GET /dashboard`) или комбинация вызовов OKR + VISION.

### Навигация

- Открывается по умолчанию при входе в приложение.
- По клику на артефакт — переход на экран коллекции артефактов с открытой карточкой этого артефакта.
- По кнопкам быстрых ссылок — переход в магазин, инвентарь, профиль, настройки.

## Экран: Профиль игрока

### Назначение

Экран для просмотра общей информации об игроке и базовой статистики. Здесь игрок видит свои достижения в цифрах и может быстро перейти к настройкам. На старте это скорее информационный экран, позже — может стать страницей с уровнями, навыками и более детальной статистикой.

### Основные элементы

- **Основная информация** — имя, аватар, дата регистрации.
- **Базовая статистика** — количество полученных артефактов (активных, завершённых, просроченных), суммарно заработанная валюта, количество покупок в магазине.
- **Краткий обзор настроек** — какие категории артефактов активны, какие типы задач из Singularity учитываются.
- **Кнопка «Настройки»** — переход к детальной настройке.

> Вопрос на будущее: нужен ли отдельный экран профиля, или достаточно карточки игрока на Dashboard и экрана настроек? На старте профиль может быть просто секцией на Dashboard, но позже, со статистикой и уровнями, отдельный экран имеет смысл.

### Навигация

- Попасть можно по клику на карточку игрока на Dashboard или через главное меню.
- Отсюда — переход на экран настроек.

## Экран: Магазин (Shop)

### Назначение

Каталог товаров, которые игрок может «купить» за внутриигровую валюту. Товары — это реальные маленькие удовольствия (вкусняшки, перерыв на сериал и т.п.) или игровые бонусы. Задача экрана — показать, что доступно, сколько стоит, и дать возможность совершить покупку с понятным подтверждением результата.

### Основные элементы

- **Каталог товаров** — список или сетка карточек товаров, каждая содержит:
  - название и описание;
  - цену в игровой валюте;
  - изображение (если есть);
  - статус наличия (в наличии, ограничено, нет в наличии).
- **Текущий баланс** — отображение доступной валюты прямо на экране магазина, чтобы игрок видел, что может себе позволить.
- **Кнопка «Купить»** — на карточке каждого товара.
- **Подтверждение покупки** — модальное окно или отдельный блок с итогом: успешно (баланс обновлён, товар добавлен в инвентарь) или отклонено (недостаточно средств).
- **Фильтры/категории** — на будущее, когда товаров станет больше.

### Навигация

- Попасть можно по кнопке «Магазин» на Dashboard или через главное меню.
- После покупки — опция перейти в инвентарь, чтобы увидеть приобретённый товар.

## Экран: Инвентарь (Inventory)

### Назначение

Список всех предметов, которые есть у игрока. Сюда попадают товары, купленные в магазине, и предметы, выданные как часть награды за завершённые артефакты. Задача экрана — показать, чем игрок владеет, в каком количестве и когда это было получено.

### Основные элементы

- **Список предметов** — каждый предмет показан с:
  - названием и типом (товар из магазина, награда за артефакт);
  - количеством;
  - датой получения;
  - пометкой, использован предмет или ещё доступен (например, «вкусняшка куплена, но ещё не съедена»).
- **Фильтры по типу** — возможность отделить «реальные» покупки (вкусняшки, удовольствия) от игровых предметов.
- **Пустое состояние** — если инвентарь пуст, подсказка, как получить первый предмет (завершить артефакт, зайти в магазин).

### Навигация

- Попасть можно по кнопке «Инвентарь» на Dashboard или через главное меню.
- Отсюда — ссылка на магазин, чтобы купить что-то ещё.

## Экран: Коллекция артефактов (Artifacts)

### Назначение

Полный список всех артефактов игрока: активных, завершённых и просроченных. Здесь игрок может увидеть свою коллекцию «трофеев» реального мира, отследить прогресс по текущим артефактам и вспомнить, что уже было заработано. Это экран-коллекция, который показывает историю достижений.

### Основные элементы

- **Список всех артефактов** — каждый артефакт показан как карточка с:
  - названием и категорией;
  - статусом (активный, завершённый, просроченный);
  - кратким прогрессом (для активных — процент, для завершённых — дата завершения).
- **Фильтры по статусу** — вкладки или переключатель: все, активные, завершённые, просроченные.
- **Карточка артефакта (детальный просмотр)** — открывается по клику на артефакт и содержит:
  - описание артефакта и правила его получения;
  - **текущую стадию** — один из четырёх этапов:
    - **Планирование** — формулировка цели артефакта, грубая декомпозиция на шаги, оценка ясности задачи, список предполагаемых ресурсов;
    - **Сбор ресурсов** — выполнение подготовительных шагов (для рабочих артефактов — интервью, документы, анализ системы; для бытовых/спортивных/учебных — свои типы подготовительных активностей);
    - **Создание артефакта** — непосредственная работа над самим артефактом (диаграмма, документ, отчёт, результат тренировки и т.п.);
    - **Оценка артефакта** — пользователь отвечает на вопросы / заполняет критерии качества, после чего система определяет редкость и итоговую награду;
  - **прогресс внутри текущей стадии** — сколько шагов пройдено на стадиях планирования, сбора ресурсов или создания;
  - детальный прогресс: общий процент, прогресс за текущий день, история шагов;
  - дедлайн и оставшееся время;
  - награду за завершение (сколько валюты, какие предметы);
  - связанные задачи Singularity (список выполненных задач, которые внесли вклад).
- На стадии «оценка» набор вопросов и критериев, которые видит пользователь, зависит от типа артефакта (рабочий, бытовой, спорт, учёба и т.д.). Визуально стадия выглядит как единый этап, но содержимое различается.

> **Источник данных:** API для артефактов `GET`/`POST` `/artifacts`.

### Навигация

- Попасть можно по кнопке «Артефакты» на Dashboard или через главное меню.
- По клику на артефакт — детальный просмотр этого артефакта.
- Из детального просмотра — ссылка на связанные задачи (если будет отдельный экран задач).

## Экран: Пассивные бафы (Привычки)

### Назначение

Экран для управления привычками и пассивными бафами игрока. Здесь игрок видит список своих привычек (зарядка, медитация, «не ел сладкое» и т.п.), их текущий уровень, стрик и окно выполнения. На этом экране игрок отмечает выполнение привычки за день — полностью или минимально — чтобы сохранить стрик, получить награду (валюта, предметы, части ресурсов) и активировать краткосрочные бафы на день, постепенно накапливая долгосрочные эффекты.

### Основные элементы

- **Список карточек привычек** — каждая карточка содержит:
  - текущий уровень привычки (например, римская цифра в углу карточки);
  - прогресс-бар по уровню (например, 5 ячеек = 5 дней; при заполнении всех — переход на следующий уровень с повышением сложности и наград);
  - текущий стрик — количество дней подряд, когда привычка соблюдалась хотя бы на минимальном уровне;
  - иконку/картинку, ассоциированную с привычкой (зарядка, медитация, кофе и т.п.);
  - окно времени выполнения (например, 8:00–9:00 для «зарядки» или 0:00–0:00 для привычек-воздержаний);
  - длительность выполнения (например, 3/5/7/10/15 минут или 24 часа для воздержаний);
  - моментальную награду за выполнение (валюта, конкретный предмет, часть ресурса вроде «10 г мате»);
  - краткосрочные бафы — текстовое описание того, что игрок получает «как человек» в течение дня (бодрость, лучшее настроение, снижение напряжения);
  - долгосрочные бафы — описание накопительного эффекта при длительном соблюдении (улучшение здоровья, координации, осанки и т.д.).
- **Элементы управления для отметки выполнения** — на каждой карточке:
  - вариант полного выполнения (например, полная зарядка);
  - вариант минимального выполнения (лайтовый вариант, который сохраняет стрик, но даёт меньше опыта и, возможно, меньшую награду).
- **Подсказка** — на карточке или внизу экрана:
  - минимальное выполнение сохраняет стрик, но даёт меньше опыта;
  - пропуск дня сбрасывает стрик.

### Навигация

- Попасть можно через главное меню или через быстрый переход с Dashboard (кнопка «Пассивные бафы» / «Привычки»).
- Отсюда — переход в профиль игрока (чтобы в будущем видеть общую статистику по привычкам и бафам) и в настройки (где можно будет включать/выключать привычки или настраивать их).

## Экран: Настройки / Интеграции (Settings)

### Назначение

Экран для управления параметрами системы и контроля интеграции с Singularity. Здесь игрок настраивает, какие типы задач и теги учитывать, видит статус синхронизации и может запустить её вручную. На старте это минималистичный экран, позже — может расшириться настройками уведомлений, правил наград и т.д.

### Основные элементы

- **Настройки задач** — выбор типов задач из Singularity, которые учитываются системой (например, только задачи с определёнными тегами).
- **Настройки артефактов** — какие категории артефактов активны, какие шаблоны использовать.
- **Статус интеграции с Singularity** — индикатор (подключено / не подключено), дата последней синхронизации, количество обработанных задач, были ли ошибки.
- **Кнопка «Синхронизировать сейчас»** — ручной запуск синхронизации.
- **Журнал ошибок** — краткий список последних ошибок синхронизации (если были).

> Вопрос на будущее: стоит ли вынести журнал синхронизаций и ошибок на отдельный экран, или оставить как секцию в настройках? На старте — секция в настройках, позже может понадобиться отдельный экран мониторинга.

### Навигация

- Попасть можно через главное меню или по кнопке «Настройки» на экране профиля.
- Отсюда — возврат на Dashboard или на любой другой экран через главное меню.

## Экран: VISION — экран визуализации (Screensaver)

### Назначение

Экран, который активируется автоматически после **3 минут** бездействия пользователя (настраивается в будущем). Служит визуальным и эмоциональным напоминанием о жизненном видении игрока и прогрессе ключевых целей.

### Основные элементы

**Основной режим (screensaver):**
- **Фон из 4 квадрантов** — картинки от каждой pinned Objective (по одной картинке из `objective_images`).
- **vision_text по центру** — текст VISION из `metadata.vision_text` (до 100 слов, полупрозрачный фон).
- **Прогресс-бары pinned Objective** — небольшой индикатор прогресса каждой из 4 закреплённых целей.

**Режим цитаты (каждые 3 минуты):**
- **purpose на весь экран** — текст «зачем» одной рандомной pinned Objective (крупный шрифт, сверху заголовок Objective).
- Через 1 минуту — автоматический возврат к основному режиму.

**Взаимодействие:**
- **Клик в любом месте** закрывает screensaver и возвращает пользователя на тот экран, где он был до активации.

> **Источник данных:** API для артефактов `GET /artifacts/vision` + API для OKR `GET /okr/pinned`.

### Обновление данных

- **MVP:** VISION загружается один раз при старте приложения и кэшируется. pinned Objective подтягиваются при активации screensaver.
- **На будущее:** Supabase Realtime — подписка на изменения `artifacts` где `category = 'VISION'` и изменения pinned Objective.

### Навигация

- Активируется автоматически по таймеру бездействия (3 минуты).
- Закрывается по клику в любом месте экрана.
- Возвращает на предыдущий экран (OKR, Dashboard и т.д.).

## Экран: Личные OKR

### Назначение

Экран для управления личными целями (Objectives) и ключевыми результатами (Key Results) игрока по 4 сферам VISION. Здесь игрок задаёт цели на период, определяет измеримые результаты и вручную обновляет прогресс. Данные OKR используются для отображения прогресс-баров по сферам рядом с VISION-экраном и в screensaver.

### Основные элементы

**Верхняя панель:**
- **Выбор квартала** — дропдаун: `Q1 2026`, `Q2 2026`, `Q3 2026`, `Q4 2026` (кварталы генерируются системой, read-only по датам).
- **Тоггл «Только активные / Все»** — переключатель между активными Objective и всеми (включая архивные).
- **Кнопка «Создать Objective»** — переход на экран создания цели.

**Основная область — 4 блока по сферам:**
- Сферы: «Тело и энергия» (`state`), «Деньги и свобода» (`finances`), «Хобби» (`hobbies`), «Любовь и близость» (`relationships`).
- У каждой сферы:
  - **Заголовок** (кликабельно → свернуть/развернуть список Objective).
  - **Индикатор прогресса сферы** — агрегированный % (среднее прогресса активных Objective в сфере).
  - **Список Objective** — карточки целей за выбранный квартал.

**Карточка Objective в списке:**
- **Название** (кликабельно → переход на детальный экран Objective + KR).
- **Purpose** (1 строка, обрезается с «…»).
- **Прогресс-бар** — агрегированный % по KR (с учётом весов).
- **Бейджи:**
  - статус (`active`, `at_risk`, `done`, `archived`, `carried_over`);
  - % выполнения (например, `65%`);
  - количество KR (например, `3 KR`).
- **Иконка pin** — если Objective закреплён для screensaver.

**Нижняя часть экрана (опционально):**
- **Блок «Ретроспектива»** — summary по выбранному кварталу: сколько Objective поставлено, сколько выполнено, средний % по сферам. Ссылка «Открыть подробную ретроспективу» → экран истории KR.

> **Источник данных:** `GET /okr/current?quarter_id=...`, `GET /okr/spheres/progress`.

### Навигация

- Попасть можно через главное меню или через кнопку «OKR» на Dashboard.
- Клик по строке Objective → детальный экран Objective (список KR, история, картинки).
- Клик по заголовку сферы → свернуть/развернуть.
- Кнопка «Создать Objective» → экран создания Objective.

## Экран: Создание/редактирование Objective

### Назначение

Форма для создания новой цели (Objective) на квартал или редактирования существующей. Форма помогает пользователю правильно сформулировать Objective, заполнить поле «зачем» (purpose) и прикрепить картинки для screensaver.

### Структура формы

**Поле: Квартал**
- Тип: дропдаун (read-only по датам).
- Значения: `Q1 2026`, `Q2 2026`, `Q3 2026`, `Q4 2026`.
- Подсказка: «Кварталы генерируются системой. Выбери период, на который ставишь цель.»

**Поле: Сфера**
- Тип: дропдаун из 4 сфер.
- Placeholder: «Выбери сферу VISION, в которой живёт эта цель».
- Значения: «Тело и энергия», «Деньги и свобода», «Хобби», «Любовь и близость».

**Поле: Название Objective**
- Тип: однострочное текстовое поле.
- Placeholder: «Научиться к концу Q2 2026 выстраивать устойчивый режим сна и восстановления так, чтобы чувствовать стабильную энергию 7/10 и выше, чтобы тянуть работу и личные проекты без выгорания».
- Подсказка под полем:
  > «Используй формулу: *Научиться к концу \<период> \<что сделать/изменить> так, чтобы \<какое новое состояние>, чтобы \<зачем это лично тебе>*. Objective должен быть амбициозным, но реалистичным, и задавать направление, а не список задач.»

**Поле: Purpose / Зачем**
- Тип: многострочное текстовое поле (textarea, 3–4 строки).
- Placeholder: «Зачем мне эта цель? Как изменится моя жизнь/состояние, если я этого достигну? Какой текст будет подбрасывать дров в костёр, когда мне тяжело?»
- Подсказка под полем:
  > «Опиши, ради чего ты это делаешь. Это текст для тебя самого, который ты будешь перечитывать в середине квартала, когда мотивация просела. Он будет показываться в screensaver как цитата.»

**Поле: Картинки Objective**
- Тип: загрузка файлов (Supabase Storage).
- Минимум: 1 картинка (обязательно, если Objective будет pinned).
- Подсказка: «Загрузи 1–5 картинок, которые ассоциируются с этой целью. Они будут показываться в screensaver, если Objective закреплён.»

**Флаг: Pinned (закрепить в screensaver)**
- Тип: чекбокс.
- Подсказка: «Закреплённые Objective (макс 4) показываются в screensaver: картинки формируют фон, purpose — как цитата. Минимум 1 картинка обязателен.»

**Кнопки:**
- «Создать Objective» / «Сохранить изменения».
- «Отмена» → возврат на экран OKR.
- «Добавить Key Results» → переход на экран создания KR (после создания Objective).

### Линк: «Как формулировать Objective?»

- Расположение: иконка ℹ️ или линк рядом с заголовком формы.
- При клике: модалка или сайдбар с мини-гайдом:
  - Формула Objective (повторяется).
  - 2–3 примера хороших Objective в разных сферах (Состояние, Финансы, Хобби).
  - Анти-примеры: «Сделать 100 тренировок» — это привычка, а не Objective.
  - Напоминание: «Objective = направление, KR = измеримый результат».

### Навигация

- Вход: кнопка «Создать Objective» на экране OKR.
- После создания: предложение «Добавить Key Results» → экран создания KR.
- Отмена → возврат на экран OKR.

## Экран: Создание/редактирование Key Result (KR)

### Назначение

Форма для добавления измеримого ключевого результата (KR) к Objective. Форма помогает выбрать правильный тип KR, задать метрику и распределить вес (weight) между KR.

### Структура формы

**Поле: Привязка к Objective**
- Тип: селектор Objective (обычно предзаполнено, если форма вызвана из контекста конкретного Objective).
- В контексте Objective: read-only, показывает название родительской цели.

**Поле: Название / формулировка KR**
- Тип: однострочное текстовое поле.
- Placeholder: «Средняя длительность сна ≥ 7:00 ч», «Пройти мед.чек-ап», «Средняя утренняя энергия ≥ 7/10».

**Поле: Тип KR**
- Тип: 3 карточки с выбором (клик по карточке раскрывает подсказку):

  **Numeric**
  > «Числовой результат: часы, деньги, количество дней, процент. Примеры: средняя длительность сна ≥ 7:00 ч; среднее 6000 шагов в день; ≥ 70% "зелёных" дней за квартал.»

  **Binary**
  > «Сделано / не сделано на уровне квартала. Примеры: пройти мед.чек-ап; завершить курс; запустить V1 проекта.»

  **Qualitative**
  > «Субъективная оценка в шкале (0–100% или 1–10). Примеры: средняя утренняя энергия ≥ 7/10; удовлетворённость работой ≥ 8/10.»

- Выбор типа влияет на набор полей ниже.

**Поля метрики (зависят от типа):**

- **Для numeric:**
  - `start_value`: «Откуда ты стартуешь в начале квартала (фактическое значение на сегодня).»
  - `target_value`: «Реалистичная, но амбициозная цель на конец квартала.»
  - `unit`: «Единицы измерения (часы сна, шаги, % "зелёных дней").»

- **Для binary:**
  - Чекбокс «Done / Not done».
  - Текстовое поле «Критерии готовности» (что считается выполненным).

- **Для qualitative:**
  - Шкала: выбор из `0–100%` или `1–10`.
  - `target_value`: целевое значение на конец квартала.

**Поле: Вес KR (weight)**
- Тип: числовое поле, %.
- Подсказка:
  > «Вес KR показывает его вклад в Objective. Сумма весов по цели должна быть = 100%. Шаг 5%. Чем выше вес, тем важнее KR для результата квартала.»
- Валидация: сумма весов всех KR Objective = 100%. При создании KR поле не обязательно, но Objective не начнёт считать прогресс, пока все KR не получат веса.

**Поле: Комментарий**
- Тип: многострочное текстовое поле (опционально).
- Placeholder: «Дополнительные критерии или заметки по этому KR».

**Кнопки:**
- «Создать KR» / «Сохранить изменения».
- «Отмена» → возврат на детальный экран Objective.
- «Распределить веса» → модалка для перераспределения 100% между всеми KR Objective.

### Инфо-блок: «Важно: KR ≠ привычка»

- Расположение: под полями формы, отдельная панель с иконкой ℹ️.
- Текст:
  > «Привычки фиксируют ежедневные действия ("лёг до 00:30", "прогулка 5000+ шагов"). Key Result фиксирует **результат за квартал** ("средняя длительность сна ≥ 7:00", "≥ 70% дней со 5000+ шагов"). Используй данные из привычек для метрик KR, но не копируй привычки один-в-один.»

### Линк: «Как формулировать KR?»

- Расположение: линк рядом с заголовком формы.
- При клике: модалка или сайдбар с мини-гайдом:
  - 2–3 примера хороших KR (numeric, binary, qualitative).
  - 1–2 примера «плохих» KR, которые на самом деле привычки.
  - Правила: «KR должен быть измеримым, с понятным критерием успешности к концу квартала.»

### Навигация

- Вход: кнопка «Добавить KR» на экране детального просмотра Objective.
- После создания: возврат на детальный экран Objective.
- Отмена → возврат на детальный экран Objective.

## Экран: Детальный просмотр Objective

### Назначение

Экран для просмотра и редактирования Objective, списка KR, управления картинками и просмотра истории изменений.

### Основные элементы

**Шапка:**
- Название Objective, сфера, квартал.
- Кнопка «Редактировать» → экран редактирования Objective.
- Флаг Pinned (иконка 📌).
- Прогресс-бар Objective (взвешенный %).

**Секция: Key Results**
- Таблица KR с полями: название, тип, метрика (`start → current → target`), прогресс-бар, вес (%), статус.
- Кнопка «Добавить KR» → экран создания KR.
- Кнопка «Распределить веса» → модалка для перераспределения 100%.
- Клик по KR → редактирование `current_value`, `weight`, `target_value`.

**Секция: Картинки Objective**
- Галерея загруженных картинок.
- Кнопка «Загрузить картинку» → файловый селектор.
- Кнопка «Удалить» на каждой картинке (запрещено, если Objective pinned и это последняя картинка).

**Секция: История изменений KR**
- Кнопка «Открыть историю» → модалка с логом событий KR (`create`, `update_value`, `update_weight`, `update_target`, `delete`, `comment`).
- Фильтры: период, тип события, конкретный KR.

**Секция: Перенос в следующий квартал**
- Кнопка «Перенести в следующий квартал» → модалка с опциями:
  - Скопировать KR? (да/нет)
  - Скорректировать веса? (да/нет)
  - В старом квартале Objective получает статус `carried_over`, в новом создаётся новый экземпляр с `parent_objective_id`.

### Навигация

- Вход: клик по Objective на экране OKR.
- Редактирование Objective → экран редактирования.
- Редактирование KR → модалка или экран редактирования KR.
- Назад → экран OKR.

## Экран: Ретроспектива квартала

### Назначение

Отдельный экран для анализа итогов квартала. Сочетает цифры (сводка, прогресс) и рефлексию (заметки, выводы). Помогает пользователю улучшить навык постановки целей.

### Основные элементы

**Секция 1: Сводка квартала (вверху)**
- **Кол-во Objective:** всего / завершены / провалены.
- **Средний % выполнения по Stretch-целям.**
- **Разбивка по сферам:** маленькие прогресс-бары/круги (4 сферы).

**Секция 2: Список Objective с итогами**
- Для каждого Objective:
  - Название, сфера, тип (Normal/Stretch/Crazy).
  - Финальный % и цветовая зона (красная < 40% / жёлтая 40–60% / зелёная > 60%).
  - Краткий вывод одной строкой («ниже ожиданий», «в целевой зоне», «превышено»).
- **Клик по Objective** → детальная карточка:
  - История изменений KR (`okr_kr_log`: target/weight/status/value).
  - Текстовый блок «Что помешало / что помогло».

**Секция 3: Рефлексия (отдельный блок)**
- 3 текстовых поля:
  - «Что получилось хорошо в этом квартале?» (`what_went_well`)
  - «Что не сработало?» (`what_didnt_work`)
  - «Что изменить в следующем квартале?» (`what_to_change`)
- Кнопка «Сохранить рефлексию» → сохраняет `retrospective_notes`.
- Индикатор: если все 3 поля заполнены → ачивка «Ретроспектива».

**Секция 4: Сравнение с прошлым кварталом (внизу/табы)**
- **Средний % по Stretch-целям:** прошлый vs текущий.
- **Кол-во проваленных Objective.**
- **Тренд по сферам:** стрелки ↑/↓.

> **Источник данных:** `GET /okr/retrospective?quarter_id=...`, `GET /okr/quarters/compare`, `POST /okr/retrospective/notes`.

### Навигация

- Вход: ссылка «Открыть подробную ретроспективу» на экране OKR (нижняя часть).
- Клик по Objective → детальная карточка с историей KR.
- Назад → экран OKR.
