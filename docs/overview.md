# Gamify Artifacts — Обзор проекта

## Цель и идея

**Gamify Artifacts** — личная система геймификации, которая превращает повседневные задачи и реальные действия в увлекательный игровой процесс. Проект интегрируется с **Singularity** (системой управления задачами) и автоматически награждает пользователя за выполнение задач: валютой, предметами и уникальными артефактами. Идея в том, чтобы рутные дела ощущались как квесты, а прогресс — как прокачка персонажа.

**Первый MVP** — личный OKR-трекер + VISION-экран. Пользователь ставит цели по 4 сферам жизни, отслеживает прогресс и видит своё жизненное видение на screensaver после минут бездействия.

## Что видит и делает пользователь

Пользователь входит в систему и видит свой профиль игрока с балансом виртуальной валюты. Он может просматривать **магазин** с товарами, покупать предметы, управлять своим **инвентарём** и коллекционировать **артефакты** — уникальные награды, привязанные к реальным выполненным задачам. Каждая выполненная задача в Singularity может стать источником награды: валюта на баланс, предмет в инвентарь или редкий артефакт. Пользователь не взаимодействует с Singularity напрямую — всё происходит через геймифицированный интерфейс.

**OKR-трекер (MVP):** пользователь ставит цели (Objectives) по 4 сферам жизни, определяет измеримые результаты (Key Results) с весами и отслеживает прогресс. Цели имеют тип (Normal/Stretch/Crazy) и поле «зачем» (purpose) — мотивационный текст, который показывается в screensaver как цитата.

**VISION-screensaver:** после 3 минут бездействия приложение показывает полноэкранный screensaver с текстом VISION и картинками закреплённых (pinned) целей. Раз в 3 минуты — цитата purpose одной из целей. Это визуальное напоминание, ради чего пользователь работает над целями.

**Ретроспектива и ачивки:** по итогам квартала пользователь анализирует результаты, заполняет заметки рефлексии и получает косметические ачивки за достижения (Stretch-цели, стрик обновлений, ретроспектива).

На **главном экране (Dashboard)** пользователь видит визуальное отображение прогресса по активным артефактам:

- **Список активных артефактов** — каждый с собственным прогресс-баром от 0 до 100%.
- **Общий прогресс по артефакту** — процент выполнения, показывающий, насколько артефакт близок к завершению.
- **Прогресс за текущий день** — отдельным цветом или меткой, сколько сделано именно сегодня по этому артефакту.
- **Оставшееся время до дедлайна** — например, «осталось 3 дня», чтобы держать фокус на сроках.

Такой Dashboard создаёт эффект видимого прогресса: игрок сразу видит, что не стоит на месте, а постоянно продвигается по своим артефактам — ключевой приём геймификации для поддержания мотивации.

## Что такое артефакт

**Артефакт** — это осязаемый результат реального действия: цифровой или физический объект, который можно сохранить, показать другим и использовать как основу для награды и прокачки. Не каждое действие порождает артефакт — только то, что имеет форму, сохраняется и может попасть в коллекцию.

**Рабочие артефакты** (фулстек-аналитика):
- Sequence diagram, BPMN-схема процесса, ERD, C4-диаграмма
- Спецификация OpenAPI, документ с требованиями, use cases
- Любой готовый документ или диаграмма — результат анализа и проектирования

**Жизненные артефакты** (продуктивность и быт):
- Запись о качественном сне из трекера
- Зафиксированная тренировка из фитнес-приложения
- Чек-лист выполненной уборки, скриншот шагомера
- Любое действие с фиксируемым цифровым следом

Артефакты — это «трофеи» реального мира, которые система превращает в игровые награды.

## Доменные модули

Архитектура построена вокруг восьми доменных модулей и слоя личных OKR:

- **Player** — профиль и данные игрока.
- **Economy** — валюта, баланс и все транзакции. Единственный модуль, который имеет право изменять кошелёк.
- **Inventory** — учёт количества предметов у игрока.
- **Shop** — каталог товаров и склад. Управляет наличием, но не трогает баланс напрямую.
- **Artifacts** — связь реальных действий с наградами, генерация и выдача артефактов.
- **Habits** — привычки как пассивные бафы и ежедневные задания (дейлики).
- **SingularityIntegration** — интеграция с API Singularity: получение задач, тегов и создание наград на основе их выполнения.
- **Личные OKR** — цели и ключевые результаты по 4 сферам VISION. Не модуль в строгом смысле, а отдельный слой над VISION: хранит Objectives и Key Results, служит источником прогресса для сфер VISION. **Важно:** OKR — про внутреннюю мотивацию («ради чего»), не выдаёт валюту/XP. Награды за OKR — только косметические ачивки.

**Разделение Habits vs OKR:** привычки фиксируют ежедневные действия («сделал / не сделал»), OKR фиксируют эффект этого поведения на уровне квартала (средние значения, проценты, субъективные оценки). OKR не превращаются во второй трекер привычек.

## Технологический стек

Проект реализован как **модульный монолит**:

- **Backend** — Supabase (PostgreSQL, Edge Functions, Storage, Auth).
- **Frontend** — статическое SPA на React + Vite + TypeScript, хостится на GitHub Pages.
- **Стейт** — Zustand, **UI** — Mantine/Chakra.
- **Внешние сервисы** — Singularity API для задач и тегов, Supabase Storage для изображений.
